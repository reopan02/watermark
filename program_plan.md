# 两阶段图像处理应用实施方案（program_plan）

## 1. 目标与范围

构建一个 **两阶段图像处理 Web 应用**，支持：

1) **步骤1：去除文字/水印**  
- 模型：`grok-4.2-image`  
- 接口范式：OpenAI 兼容 `/v1/chat/completions`  
- Prompt：`Remove all watermarks and text`

2) **步骤2：补全画面（全画幅扩展 + 清理冗余元素）**  
- 模型：`gemini-3-pro-image-preview`  
- 默认输出分辨率：2K（建议 `2048x2048` 或按比例最大边 2048）  
- Prompt：`全画幅展示图像，去除多余元素`

3) 通用能力  
- 结果预览  
- 下载保存  
- 步骤1结果可“一键进入步骤2”快速处理

---

## 2. 统一技术架构

### 2.1 架构原则

- **统一输入协议**：所有图像上传到网页后统一为 `data:{mime};base64,{payload}`
- **统一调用底座**：LangChain 封装多模型调用（Grok / Gemini）
- **统一任务流**：前端只关心“创建任务 → 获取结果”

### 2.2 分层设计

#### 前端层（Web）
- 图片上传（拖拽/点击）
- 两阶段处理入口
- 预览、下载、步骤联动

### 流程 A：去除文字（步骤1）
1. 用户上传图像  
2. 前端转 base64  
3. 点击“去除文字”  
4. 后端创建 `task_1`，编排到 Grok（OpenAI 格式）  
5. 返回结果图  
6. 前端展示预览 + 下载 + “继续补全画面”

### 流程 B：补全画面（步骤2）
1. 用户上传图像或使用步骤1输出  
2. 点击“补全画面”  
3. 后端创建 `task_2`，编排到 Gemini  
4. 返回补全结果  
5. 前端预览并下载

### 流程 C：步骤联动（关键）
1. 步骤1完成后显示按钮“继续补全画面”  
2. 点击后自动将步骤1输出填充到步骤2输入区  
3. 自动带上 `source_task_id=task_1`，可一键执行步骤2
